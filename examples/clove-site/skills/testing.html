Test the risks that matter.

Use this Agent Skill to plan and operate risk-based testing across units, integrations, contracts, browsers, accessibility, releases, production, and recovery.

A green pipeline reduces uncertainty only within the behavior and conditions it actually asserts.

Match each failure risk to credible evidence.

This skill chooses the lowest test layer that can establish confidence without confusing test count or coverage with product quality.

When to use this skill

You are defining test strategy, cases, CI gates, coverage, fixtures, compatibility, migrations, progressive delivery, rollback, defect evidence, or release confidence.

How this skill works

  1. Build a change-risk packet around behavior, dependencies, data, and possible harm.
  2. Map each material risk to the test layer that can expose it credibly.
  3. Specify preconditions, actions, outcomes, side effects, and diagnostic evidence.
  4. Control sources of flakiness and design a fast, high-signal pipeline.
  5. Verify release, rollback, repair, and the lesson from escaped failures.

Trust the evidence

  • Test public behavior and policy, not incidental implementation.
  • Do not mock the boundary whose real semantics carry the risk.
  • Separate accessibility, security, privacy, performance, and recovery evidence.
  • Report exact checks, results, coverage limits, and untested risks.
