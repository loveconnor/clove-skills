Security and Privacy Agent Skill Design for harm, defense, and recovery.

Use this Agent Skill to design secure, privacy-respecting systems by modeling threats and affected people, selecting explicit controls, and requiring evidence.

Security and privacy overlap, but they are not interchangeable.

Make trust boundaries and safe failure visible.

This skill connects assets, people, threats, personal data, identity, authorization, dependencies, detection, incidents, and recovery.

When to use this skill

A change affects authentication, sessions, authorization, tenants, personal data, secrets, dependencies, logging, vulnerabilities, incidents, backups, or recovery.

How this skill works

  1. Map actors, assets, data, purposes, environments, dependencies, and trust boundaries.
  2. Write causal threat, privacy-risk, and abuse scenarios.
  3. Select preventive, detective, responsive, and recovery controls.
  4. Specify identity decisions and engineer the complete personal-data lifecycle.
  5. Verify evidence, prepare incident response, and exercise recovery.

Non-negotiable

  • Validate untrusted data and enforce authorization at trusted boundaries.
  • Keep secrets and unnecessary personal data out of source, URLs, logs, and analytics.
  • Use approved cryptography and managed secret lifecycles.
  • Never claim compliance, encryption, restore, or audit evidence that was not established.
