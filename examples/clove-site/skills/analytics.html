Analytics Agent Skill Measure what can change a decision.

Use this Agent Skill to design, implement, audit, and interpret product analytics around observable behavior, governed definitions, privacy, and uncertainty.

Start with a decision, then collect the minimum data needed to answer it.

Treat analytics as a versioned data product.

Reliable measurement makes event meaning, identity, eligibility, consent, data quality, and interpretation explicit.

When to use this skill

You are defining metrics, events, identity, funnels, cohorts, retention, experiments, dashboards, instrumentation, guardrails, or data-quality monitoring.

How this skill works

  1. Frame the decision, population, behavior, time horizon, mechanism, and guardrails.
  2. Model entities and write governed event contracts.
  3. Instrument at the boundary that can know the claimed outcome.
  4. Define identity, eligibility, consent, and analysis before reading results.
  5. Verify quality and interpret description, association, attribution, and causation separately.

Keep explicit

  • Event trigger, non-trigger cases, version, ownership, and prohibited data.
  • Anonymous, authenticated, account, tenant, session, and experiment identity.
  • Deduplication, ordering, delayed delivery, reversals, and exclusions.
  • What the available data can and cannot establish.
