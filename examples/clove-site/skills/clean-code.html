Write code that can be owned.

Use this Agent Skill to implement and review code that a team can understand, verify, change, operate, and remove.

Correctness, safety, accessibility, compatibility, and user-visible behavior outrank elegance.

Keep the main path clear and the important risks visible.

Maintainable code makes contracts, state, dependencies, failures, and ownership explicit.

When to use this skill

You are creating, changing, reviewing, or removing application code, APIs, tests, configuration, data models, dependencies, error paths, or performance behavior.

How this skill works

  1. State the user or system outcome, non-goals, risks, and acceptance evidence.
  2. Find the current owner and characterize unfamiliar behavior.
  3. Define observable contracts, invariants, and state transitions.
  4. Make the smallest clear change with behavior-focused tests.
  5. Read the diff and run the project’s real checks before review.

Code rules

  • Name domain roles, ownership, units, and state clearly.
  • Keep I/O, side effects, validation, and errors visible.
  • Add abstractions only for real concepts or repeated knowledge.
  • Keep secrets and unnecessary personal data out of code and logs.
