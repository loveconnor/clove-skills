Color Agent Skill Give color a job.

Use this Agent Skill to design and audit semantic color systems, themes, states, charts, and contrast across real interface conditions.

Begin with roles, not favorite hues.

Build relationships, not swatches.

Color needs to identify actions, status, hierarchy, and product character across real content and settings.

When to use this skill

You are creating palettes or tokens, mapping a brand into UI, adding themes, evaluating contrast, or designing status and data visualizations.

How this skill works

  1. Inventory existing colors and the roles they currently serve.
  2. Define semantic roles for surfaces, text, actions, borders, focus, and status.
  3. Construct primitive families, then map roles to states and themes.
  4. Measure approved foreground/background pairs.
  5. Test real components, media, forced colors, and color-vision differences.

Accessibility floor

  • Normal text needs 4.5:1 contrast.
  • Required UI boundaries, focus, and meaningful graphics need 3:1 contrast.
  • Never rely on color as the only signal.
  • Design dark and high-contrast themes by role, not inversion.
